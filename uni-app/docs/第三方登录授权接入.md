# 第三方登录授权接入说明

对脉小程序提供独立登录页 `/pages/login/login`，供**第三方小程序**唤起并完成微信头像、昵称授权。授权数据在**本地收集**，**不请求对脉服务端**；完成后 Toast「登录成功」并关闭当前小程序，将结果回传给调用方。

---

## 一、页面信息

| 项 | 说明 |
|---|---|
| 路径 | `/pages/login/login` |
| 可选参数 | `state`：调用方自定义关联 ID，原样透传回 `extraData` |
| 示例路径 | `/pages/login/login?state=order-20260329-001` |

> 本页**不需要**赛事 `eventGuid`，与选手页内弹窗登录（`MpWechatLoginSheet`）相互独立。

---

## 二、第三方唤起示例

### 1. 小程序跳转（推荐）

调用方在按钮点击等用户手势中执行：

```javascript
wx.navigateToMiniProgram({
  appId: 'wx对脉小程序AppID', // 替换为实际 AppID
  path: '/pages/login/login?state=your-correlation-id',
  envVersion: 'release', // develop | trial | release
  success() {
    console.log('已跳转对脉授权页')
  },
  fail(err) {
    console.error('跳转失败', err)
  },
})
```

**接收回传数据**（在调用方 `App.onShow` 或页面 `onShow`）：

```javascript
App({
  onShow(options) {
    const extra = options?.referrerInfo?.extraData
    if (!extra || extra.ok !== true) return

    console.log('授权 code', extra.code)
    console.log('昵称', extra.nickName)
    console.log('头像', extra.avatarUrl)
    console.log('state', extra.state)

    // 将 extra.code 发往自有服务端，换取 openId 并签发 token
  },
})
```

### 2. 小程序码 / URL Link

生成指向登录页的小程序码或 URL Link，路径示例：

```
pages/login/login?state=your-correlation-id
```

用户扫码或点击链接进入后，流程与上文一致。若**非**从其他小程序 `navigateToMiniProgram` 唤起，授权成功后本小程序调用 `exitMiniProgram` 关闭；此时调用方无法通过 `referrerInfo.extraData` 接收数据，**仅适用于小程序间跳转场景**。

### 3. 开发者工具联调

1. 微信开发者工具 → **添加编译模式**
2. 启动页面：`pages/login/login`
3. 启动参数：`state=test123`（可选）
4. 编译运行

> 开发者工具内单独打开时，`exitMiniProgram` 表现可能与真机不一致；完整回传链路请在**真机 + 第三方小程序跳转**下验证。

## 三、授权流程

```
第三方小程序 navigateToMiniProgram
        ↓
对脉 /pages/login/login 打开
        ↓
（可选）本地已有资料 → 静默 wx.login 取 code → 直接回传
        ↓
微信隐私协议确认（若需要）
        ↓
getUserProfile 获取昵称、头像
        ↓
wx.login 获取临时 code
        ↓
Toast「登录成功」（约 1.5s）
        ↓
navigateBackMiniProgram({ extraData })  // 有来源小程序
或 exitMiniProgram()                    // 无来源小程序
```

## 四、输出数据（extraData）

### 授权成功

| 字段 | 类型 | 说明 |
|---|---|---|
| `ok` | `boolean` | 固定为 `true` |
| `code` | `string` | `wx.login` 临时凭证，**5 分钟有效、一次性** |
| `loginCode` | `string` | 与 `code` 相同，兼容字段 |
| `nickName` | `string` | 微信昵称 |
| `avatarUrl` | `string` | 微信头像 URL |
| `state` | `string` | 唤起时传入的 `state` 原样透传 |

**JSON 示例：**

```json
{
  "ok": true,
  "code": "0f3xxxxxxxxxxxxxxxxx",
  "loginCode": "0f3xxxxxxxxxxxxxxxxx",
  "nickName": "张三",
  "avatarUrl": "https://thirdwx.qlogo.cn/mmopen/...",
  "state": "order-20260329-001"
}
```

### 用户拒绝授权

| 字段 | 类型 | 说明 |
|---|---|---|
| `ok` | `boolean` | `false` |
| `state` | `string` | 透传 state |
| `errMsg` | `string` | 固定为 `user_denied` |

**JSON 示例：**

```json
{
  "ok": false,
  "state": "order-20260329-001",
  "errMsg": "user_denied"
}
```

## 五、第三方服务端：用 code 换取 openId

对脉登录页**不返回** `openId` / `token`（需在服务端用 code 换取）。第三方自有后端示例：

**请求微信接口：**

```
GET https://api.weixin.qq.com/sns/jscode2session
    ?appid=APPID
    &secret=SECRET
    &js_code=CODE
    &grant_type=authorization_code
```

**响应示例：**

```json
{
  "openid": "oXXXX",
  "session_key": "xxx",
  "unionid": "uXXXX"
}
```

> `appid` / `secret` 须为**对脉小程序**在微信公众平台上的凭证；若第三方与对脉非同一主体，需使用微信开放平台绑定后的 `unionid` 做账号打通。

第三方在换取 `openid` 后，自行签发业务 `token` 返回给客户端。

## 六、注意事项

1. **回传前提**：调用方须通过 `wx.navigateToMiniProgram` 唤起，才能在 `onShow` 的 `referrerInfo.extraData` 中收到数据。
2. **code 时效**：`code` 仅可使用一次，收到后应尽快发往服务端换 `openid`。
3. **静默授权**：用户曾授权且本地有资料时，可能不再弹 `getUserProfile`，但仍会重新 `wx.login` 获取新 `code`。
4. **隐私合规**：对脉小程序须在微信公众平台配置「收集昵称、头像」等隐私类型；调用方亦需遵守各自隐私政策。
5. **与选手页登录区分**：赛事选手页仍使用 `MpWechatLoginSheet` 弹窗 + 对脉服务端 `wx-login`（需 `eventGuid`），与本独立登录页无关。
